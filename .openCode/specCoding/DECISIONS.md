## 已采纳的决策

### ADR-001: 使用 Pinia 作为状态管理方案

**日期**: 2026-02-15  
**状态**: ✅ 已采纳

**背景**:
项目需要管理学习集列表、当前 PDF、用户设置等全局状态，需要在组件间共享数据。

**决策**:
使用 Pinia 作为官方推荐的状态管理库，替代 Vuex。

**原因**:
1. Vue 3 官方推荐
2. TypeScript 支持完善
3. API 简洁，学习曲线低
4. 支持模块化
5. 体积小巧（~1KB）

**后果**:
- ✅ 状态管理统一
- ✅ 组件间通信简化
- ✅ 调试工具支持好
- ⚠️ 需要学习 Pinia API

**替代方案**:
1. Vuex - 较重，API 复杂
2. 纯 Composition API - 不适合复杂状态
3. Zustand - 第三方库，生态较小

### ADR-002: 使用思源块属性作为主要数据存储

**日期**: 2026-02-15  
**状态**: ✅ 已采纳

**背景**:
需要持久化学习集、标注、思维导图等数据，需要支持跨设备同步。

**决策**:
使用思源笔记的块属性系统作为主要数据存储，localStorage 作为临时/缓存存储。

**原因**:
1. 思源笔记原生支持
2. 支持跨设备同步
3. 数据与文档绑定
4. API 成熟稳定

**后果**:
- ✅ 数据可随思源文档同步
- ✅ 支持跨设备
- ✅ 用户数据可控
- ⚠️ 依赖思源 API
- ⚠️ 需要处理临时 ID 映射

**替代方案**:
1. 纯 localStorage - 不支持跨设备
2. IndexedDB - 复杂，不同步
3. 云端存储 - 增加复杂度

### ADR-003: 采用 MarginNote4 风格的左右分栏布局

**日期**: 2026-03-01  
**状态**: ✅ 已采纳

**背景**:
用户希望实现类似 MarginNote4 的左 PDF 右思维导图学习体验，提高学习效率。

**原因**:
1. MarginNote4 已验证的学习模式
2. PDF 与思维导图实时联动
3. 用户习惯的布局
4. 提供多种选择

**后果**:
- ✅ 学习效率提升
- ✅ 用户体验优化
- ✅ 功能差异化
- ⚠️ 布局复杂度增加
- ⚠️ 需要处理同步逻辑

**实现细节**:
- 思维导图默认宽度：400px
- 可调节范围：280px - 600px
- 使用可拖拽分隔条
- 实时同步延迟：500ms 防抖

### ADR-006: 使用临时 ID 支持未保存到思源的思维导图

**日期**: 2026-03-01  
**状态**: ✅ 已采纳

**背景**:
思维导图创建时需要先有块 ID 才能保存，但创建思维导图块又需要 ID，存在循环依赖。

**决策**:
使用临时 ID（格式：`temp-{时间戳}-{随机数}`）支持未保存到思源的思维导图，数据存储在 localStorage。

**原因**:
1. 解决循环依赖
2. 用户体验流畅
3. 实现简单
4. 可后续迁移到正式 ID

**后果**:
- ✅ 用户体验优化
- ✅ 技术债务可控
- ⚠️ 需要处理 ID 迁移
- ⚠️ localStorage 限制

**实现细节**:
```typescript
// 临时 ID 格式
const tempId = `temp-${Date.now()}-${Math.random().toString(36).substring(2, 8)}`

// 验证临时 ID
function isTempId(blockId: string): boolean {
  return blockId.startsWith('temp-')
}
```

### ADR-007: 实现 PDF 与思维导图实时同步

**日期**: 2026-03-01  
**状态**: ✅ 已采纳

**背景**:
MarginNote4 的核心体验是 PDF 摘录自动同步到思维导图，需要实现类似功能。

**决策**:
使用事件总线实现 PDF 与思维导图的实时同步：
1. PDF 端创建摘录后触发 `annotation-created` 事件
2. 思维导图端监听事件并创建对应节点
3. 建立双向关联（nodeId ↔ annotationId）

**原因**:
1. MarginNote4 核心功能
2. 提升学习效率
3. 事件驱动解耦
4. 技术实现可行

**后果**:
- ✅ 用户体验质的提升
- ✅ 功能竞争力增强
- ⚠️ 同步逻辑复杂
- ⚠️ 需要处理性能问题

**实现细节**:
- 事件：`annotation-created`, `annotation-updated`, `annotation-deleted`
- 防抖：500ms
- 双向关联：node.data.annotationId
