## 🚀 分阶段实施路线图

### 阶段一：基础插件框架搭建（1-2天）
*   **目标**：搭建开发环境，创建一个能在思源中正常显示、打开、关闭的基础插件面板。
*   **核心实现方案**：
    1.  配置 WebStorm 运行/调试配置。
    2.  在 `src/index.ts` 中注册思源顶栏按钮。
    3.  创建一个 Vue 组件 (`App.vue`) 作为插件主面板，并通过 `createApp` 挂载到页面。
*   **关键产出**：一个可通过顶栏图标开关的空白侧边栏面板。

### 阶段二：PDF文件选择与基础渲染（3-5天）
*   **目标**：实现从思源中选择 PDF 文件，并在插件面板内渲染显示。
*   **核心实现方案**：
    1.  调用 `/api/filetree/listDocTree` 或 `/api/query/block` 获取当前文档下的 PDF 资源路径。
    2.  使用 `pdfjs-dist` 库初始化 PDF 查看器。
    3.  实现基础的翻页、缩放功能。
*   **关键产出**：一个简易的 PDF 阅读器，嵌入在思源插件面板中。

### 阶段三：PDF标注提取与数据化（1-2周，核心难点）
*   **目标**：捕获 PDF 上的标注，并提取出结构化数据（内容、颜色、坐标）。
*   **核心实现方案（推荐路线A）**：
    1.  利用思源原生 PDF 标注功能，标注会自动生成 `NodeTextMark` 类型的块。
    2.  通过 `/api/block/getBlockKramdown` 接口，获取这些块的详细信息，解析其中的 `file-annotation-ref` 属性，得到标注内容、页码和坐标矩形 `rect`。
    3.  将解析后的数据存储为前端状态。
*   **关键产出**：一个能解析当前 PDF 所有标注信息的函数，并输出结构化的 JSON 数据。

### 阶段四：结构化思维导图生成（3-5天）
*   **目标**：将线性的标注数据转换为有层级的思维导图。
*   **核心实现方案**：
    1.  设计组织逻辑（如按颜色分类、按页码排序）。
    2.  将组织好的数据转换为 Markdown 字符串。
    3.  使用 `markmap-lib` 和 `markmap-view` 将 Markdown 渲染为交互式思维导图。
*   **关键产出**：一个根据 PDF 标注自动生成的思维导图组件。

### 阶段五：双向交互与编辑（1-2周）
*   **目标**：实现导图节点与 PDF 视图的联动，以及内容的编辑同步。
*   **核心实现方案**：
    1.  为导图节点绑定 `click` 事件，点击时调用 PDF.js 的滚动方法，定位到对应标注位置。
    2.  在导图中集成编辑功能，修改后调用 `/api/block/updateBlock` 更新思源中的块内容。
*   **关键产出**：一个可交互、可编辑的思维导图，与 PDF 阅读器实时联动。

### 阶段六：优化、打包与发布（1周）
*   **目标**：完善用户体验，准备发布。
*   **核心实现方案**：
    1.  性能优化（虚拟滚动、防抖节流）。
    2.  UI/UX 美化（适配思源主题色、图标设计）。
    3.  配置 `release.js`，生成 `package.zip`，编写 `README`，提交到集市。
*   **关键产出**：一个稳定、好用的思源笔记插件，可供他人下载使用。
